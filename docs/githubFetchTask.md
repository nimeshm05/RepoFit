# **Requirements to get Github repositories:**

1. Fetch only public, non-archived GitHub repositories through the GitHub API.
2. Retrieve repositories with a meaningful, non-empty description that clearly explains the project purpose. The description should be long enough and not 2-3 sentences.
3. Fetch repositories that contain essential metadata including README, primary language, license, topics, stars, and last updated date.
4. Prioritize repositories with recent activity and exclude inactive or abandoned projects.
5. Retrieve contribution-related signals such as issue availability, contribution guidelines, and beginner-friendly labels where available.
